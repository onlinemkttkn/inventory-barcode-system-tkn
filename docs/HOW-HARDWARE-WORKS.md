# การทำงานของระบบ Hardware

## Auto Mode

POS ไม่เดาว่าโปรแกรมใดกำลังรัน แต่ตรวจ `/health` ก่อนทุกครั้ง:

- `tkn-hardware-service` ที่ 17890 = ตัวหลัก
- `tkn-rongta-bridge` ที่ 17891 = ตัวสำรอง
- Browser Print = ทางเลือกสุดท้ายสำหรับการพิมพ์

## การเปิดลิ้นชัก

หลัง RPC `create_pos_sale` สำเร็จและวิธีชำระเป็น CASH เท่านั้น POS จึงเรียก
`TKNHardware.openDrawer()` หากคำสั่งล้ม ระบบแจ้งเตือน แต่ไม่ย้อนบิลและไม่ตัด
สต็อกซ้ำ

## การพิมพ์

ใบเสร็จ HTML ภาษาไทยและ CSS 80 มม. ใช้ Browser Print เป็นค่าเสถียร เนื่องจาก
การพิมพ์ RAW ต้องยืนยัน Code Page ภาษาไทยของ Rongta แต่ละรุ่นก่อน ส่วน
`/print/raw` ยังคงมีไว้สำหรับ ESC/POS payload ที่ผ่านการทดสอบแล้ว

## การตั้งค่า

ค่าฮาร์ดแวร์เป็นค่าประจำเครื่อง จึงเก็บใน `localStorage` ของ Browser เครื่อง
แคชเชียร์ ไม่เก็บรวมในฐานข้อมูล เพื่อไม่ให้สาขาหนึ่งเปลี่ยนเครื่องพิมพ์ของอีกสาขา
